Folie 1

Common neural codes in biology: Rate code

Rate code: Information steckt in der Feuerrate. Beispiel: V1-Tuning-Kurven — die Feuerrate eines Neurons hängt von der Balkenorientierung ab (Peak bei der bevorzugten Orientierung).

Aber: Rate-Codes leiden oft unter einem langen Integrationsfenster, um zuverlässige Schätzungen zu erhalten.

Beleg: V1-Physiologie, Orientierungsselektivität (Hubel & Wiesel 1968) — die neuronale Antwort (spikes/s) folgt einer glockenförmigen Kurve über der Stimulus-Orientierung (Peak um 0°).

Erklärung: Der Rate-Code ist der einfachste und robusteste Code, aber langsam: Man muss über ein Zeitfenster mitteln, um die Rate genau zu kennen. Das motiviert die schnelleren Codes auf den folgenden Folien.
Folie 2

On-/Off-center receptive fields (retinale Ganglienzellen)

Klassisches Center-Surround-Schema: On-center-Zellen feuern maximal bei Licht im Zentrum, Off-center-Zellen bei Licht im Umfeld. Diffuse Beleuchtung (A) erzeugt wenig Antwort, ein kleiner Lichtpunkt im Zentrum (B/C) treibt die On-Zelle stark, Umfeld-Beleuchtung (E) treibt die Off-Zelle.

Erklärung: Das antagonistische Center-Surround-Feld lässt sich mathematisch als Difference of Gaussians (DoG) beschreiben. Es implementiert eine Kontrast-/Kantendetektion — die Retina überträgt vor allem Änderungen, nicht die absolute Helligkeit.
Folie 3

Place code & temporal code

Place code

Die Lage/Identität des Neurons kodiert die Information. Beispiel: die retinotope Karte in V1.

Temporal code

Das relative Timing der Spikes kodiert Information. Beispiel: binaurale Schall-Lokalisation der Schleiereule (barn owl).

  • Präzision: ca. 1°–2°
  • Zeitliche Auflösung: ca. 10 µs  ↔  Membran-Zeitkonstante: ca. 10 ms!
Erklärung: Der auffällige Punkt: Die Eule erreicht eine zeitliche Auflösung von 10 µs, obwohl die Membran-Zeitkonstante 1000× größer ist (10 ms). Ein einzelnes Neuron kann das nicht — es braucht ein cleveres Netzwerk (Koinzidenzdetektion, Folie 6).
Folie 4

Time-to-first-spike (latency code) & phase code

Time-to-first-spike / Latenz-Code

Das Timing des Feuer-Onsets kodiert Information (Beispiel: Salamander-Retina).

Einfaches Spielzeug-Netzwerk für Single-Spike-Latenz-Encoding: Ein Eingangsstrom \(I(t)\) treibt ein exzitatorisches Neuron \(e\), das ein inhibitorisches Neuron \(i\) ansteuert. Bei schwachem Input steigt \(u_e\) langsam → großes \(\Delta t\) bis zum Spike; bei starkem Input steigt \(u_e\) schnell → kleines \(\Delta t\).

Phase code

Relatives Timing zu einem oszillatorischen Signal (z. B. Theta-Rhythmus). Beispiel: Place cells im Hippocampus — grob ein Place-/Rate-Code, fein ein Phase-Code (Phasen-Präzession).

Erklärung: Latenz-Code = „stärkerer Reiz → früherer Spike“. Das ist viel schneller als ein Rate-Code, weil schon der erste Spike Information trägt. Der Phase-Code nutzt zusätzlich die Position innerhalb einer Hintergrund-Oszillation als Informationsträger.
Folie 5

Place cells (Nobelpreis Medizin 2014)

Räumliche Feuermuster von 8 Place-Cells aus der CA1-Schicht einer Ratte. Die Ratte lief auf einer Bahn hin und her; jeder Punkt markiert eine Position, an der ein Aktionspotential auftrat, die Farbe kennzeichnet das jeweilige Neuron. Jede Zelle feuert nur an einem bestimmten Ort (ihrem „place field“).

Erklärung: Beispiel für einen Place-Code: Welche Zelle feuert, verrät den Ort des Tieres. Die Entdeckung des „inneren GPS“ (place cells / grid cells) wurde 2014 mit dem Nobelpreis ausgezeichnet (O'Keefe, Moser & Moser).
Folie 6

Barn owl: binaurale Schall-Lokalisation

Mechanismus: Der Schall erreicht das linke und rechte Ohr mit einer Laufzeitdifferenz (ITD). Axone von beiden Seiten bilden Delay Lines (Verzögerungsleitungen), die auf eine Reihe von Koinzidenzdetektoren treffen. Genau der Detektor, bei dem sich die verzögerten Signale gleichzeitig treffen, feuert — seine Position kodiert die Schallrichtung.

Erklärung: Dies ist das Jeffress-Modell. Es löst das Paradox von Folie 3: Ein einzelnes Neuron ist zu träge (10 ms), aber ein Netzwerk aus Delay Lines + Koinzidenzdetektoren erreicht µs-Genauigkeit. Timing (temporal code) wird hier in einen Ort (place code) übersetzt.
Folie 7

Firing-rate-based and population models

Spikende Neuronen zu modellieren ist rechenaufwändig und analytisch schwierig.

Idee: Ersetze Neuronen (oder kleine Populationen) durch „firing-rate units“:

  • ＋ gewinnt analytische Handhabbarkeit
  • － verliert spike-timing-basierte Codes!

Rate-based input integration

Netzwerk-Motiv: Ein postsynaptisches Neuron mit Ausgangsrate \(z(t)\) erhält Eingangsraten \(x_i(t)\) über Gewichte \(w_i\):

  • \(w_i>0\): Erregung (excitation)
  • \(w_i<0\): Hemmung (inhibition)
Erklärung: Der große Übergang dieser Vorlesung: weg von einzelnen Spikes, hin zu kontinuierlichen Feuerraten. Das ist die Brücke zu künstlichen neuronalen Netzen — Rate-Units sind im Kern die „Neuronen“ des Deep Learning.
Folie 8

Rate-based input integration – Herleitung

Eingangsstrom \(I(t)\) zum postsynaptischen Neuron mit exponentiellem CUBA-Kernel. Man ersetzt den Spike-Train durch seinen Erwartungswert (die Rate \(x_i(\tau)\)):

\[ I(t) = \sum_i w_i\sum_s \varepsilon(t-t_s^{i}) = \sum_i w_i\int_{-\infty}^{t}\!\varepsilon(t-\tau)\,\rho_i(\tau)\,d\tau \;\longrightarrow\; \sum_i w_i\int_{-\infty}^{t}\!\varepsilon(t-\tau)\,x_i(\tau)\,d\tau \]
\[ = \sum_i w_i\int_{-\infty}^{t}\underbrace{\Theta(t-\tau)}_{=1}\,e^{-\frac{t-\tau}{\tau_{syn}}}\,x_i(\tau)\,d\tau = \sum_i w_i\,e^{-t/\tau_{syn}}\int_{-\infty}^{t} e^{\tau/\tau_{syn}}\,x_i(\tau)\,d\tau \]

Ableiten nach \(t\) liefert eine Differentialgleichung:

\[ \tau_{syn}\,\dot I = -I + \tau_{syn}\,\vec w\cdot\vec x \]

Steady state (\(\dot I=0\)): \(\quad I = \tau_{syn}\,\vec w\cdot\vec x\).

Erklärung: Aus der Faltung mit dem exponentiellen Kernel wird eine einfache lineare DGL: Der synaptische Strom \(I\) folgt der gewichteten Eingangssumme \(\vec w\cdot\vec x\) wie ein Tiefpassfilter mit Zeitkonstante \(\tau_{syn}\).
Folie 9

Rate-based response & Aktivierungsfunktionen

  • Bei konstantem Input \(I\) läuft die Ausgangsrate gegen \(\displaystyle\lim_{t\to\infty} z(t) = F(\vec w\cdot\vec x)\); dabei ist \(\tau_{syn}\) in die Response-Funktion \(F\) absorbiert.
  • Bei zeitvariablem Input folgt der Output mit etwas Verzögerung (Membran-Zeitkonstante, Populationsdynamik):
\[ \tau\,\dot z = -z + F(\vec w\cdot\vec x) \]

\(\tau\) berücksichtigt die Tiefpassfilterung von Strom und Antwort. Gängige Response-/Aktivierungsfunktionen \(F(I)\):

  • Softplus: \(\log(1+e^{I})\)
  • Sigmoid: \(z_{max}\,\dfrac{1}{1+e^{-I}}\)
  • Lineare Gleichrichtung (ReLU): \([I]_+ = \max(0,I)\)
  • Exponentiell: \(e^{I}\)
Erklärung: \(F(I)\) ist die f-I-Kurve (Rate als Funktion des Inputs). Genau diese Funktionen sind die Aktivierungsfunktionen künstlicher neuronaler Netze — Sigmoid und ReLU kennt man aus dem Deep Learning.
Folie 10

Rate-based neural network models

Zwei Neuronenschichten mit rekurrenten und Feed-Forward-Verbindungen:

  • Rekurrentes Gewicht \(w_{kj}\) (Konvention \(w_{\text{post},\text{pre}}\))
  • Netzwerk-Population \(z_k(t)\)
  • Effektiver Input \(V_k = \sum_i v_{ki}\,x_i\)
  • Forward-Gewicht \(v_{ki}\), Input \(x_i(t)\)

Populationsdynamik

\[ \tau\,\dot z_k = -z_k + F\!\left(\sum_{j=1}^{K} w_{kj}\,z_j + \sum_{i=1}^{N} v_{ki}\,x_i + b_k\right) \]

\(F\) = Gain-Funktion, \(b_k\) = Bias bzw. intrinsische Erregbarkeit. Man startet einfach und erhöht dann die Komplexität.

Erklärung: Das ist die allgemeine Gleichung eines rekurrenten neuronalen Netzes: Feed-Forward-Input \((v_{ki}x_i)\) plus Rückkopplung \((w_{kj}z_j)\) plus Bias, durch die Nichtlinearität \(F\) geschickt. Setzt man \(w_{kj}=0\), erhält man ein reines Feed-Forward-Netz (nächste Folie).
Folie 11

Feed-forward networks · stationary binary logic

Feed-Forward = keine rekurrenten Verbindungen (\(w_{ij}=0\)).

Warmup: stationäre binäre Logik. Seien \(x_i\in\{L(\text{ow}),H(\text{igh})\}\) mit \(0\le L

Beispiel 1: NOT(x)

Ein Neuron mit Gewicht \(-1\) und Bias \(H+L\):

\[ z = [-1\cdot x + H + L]_+ = \begin{cases} H & \text{falls } x=L\\ L & \text{falls } x=H \end{cases} \]
Erklärung: Mit nur einem Gewicht und einem passenden Bias realisiert ein Rate-Neuron die logische NOT-Funktion: aus Low wird High und umgekehrt. Das zeigt, dass solche Netze diskrete Logik ausführen können.
Folie 12

Beispiel 2: NAND & Turing-Vollständigkeit

Ein zweistufiges Netz realisiert NAND(\(x_1,x_2\)): \(x_1,x_2\) mit Gewicht \(+1\) auf \(z_1\) (Bias \(-(H+L)\)), dann \(z_1\) mit Gewicht \(-1\) auf \(z_2\) (Bias \(H\)).

\(x_1\backslash x_2\)LH
LHH
HHL
  • NAND ist eine nichtlineare Funktion — man nutzt die Nichtlinearität von \([\,\cdot\,]_+\) an \(z_1\): für NAND(H,H) ist \(2H-(H+L)=H-L>0\), für die anderen Fälle wird abgeschnitten.
  • Alle binären logischen Ausdrücke lassen sich durch NAND darstellen (funktionale Vollständigkeit), z. B. \(\text{NOT}(x)=\text{NAND}(x,x)\).
  • Zudem: Addition, Multiplikation realisierbar.
→ Rate-based neural networks können (im Prinzip) jede CPU-Instruktion emulieren → Turing-vollständig.
Erklärung: Der krönende Abschluss: Weil NAND funktional vollständig ist und ein kleines Rate-Netz NAND berechnet, kann ein hinreichend großes Rate-basiertes Netz beliebige Berechnungen ausführen. Neuronale Netze sind also universelle Rechenmaschinen.
