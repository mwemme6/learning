Folie 1

Tuning curves

Ein Neuron \(z_k\) erhält Input \(\vec x=(x_1,\dots,x_N)\) über die Gewichte \(\vec V_k=(V_{ki})_{1\dots N}\). Für statischen Input (\(\vec x(t)=\text{const}\)) gilt \(z_k\to F(\vec V_k\cdot\vec x)\) für \(t\to\infty\).

Frage: Auf welchen Stimulus \(\vec x^{*}\) reagiert Neuron \(z_k\) am stärksten?

Da \(F(\cdot)\) (streng) monoton steigend ist:

\[ \vec x^{*} = \arg\max_{\vec x}\ \vec V_k\cdot\vec x = \arg\max_{\vec x}\ \|\vec x\|\,\|\vec V_k\|\,\cos\big(\angle(\vec V_k,\vec x)\big) \]

Bei fester Stimulusstärke (\(\|\vec x\|=\text{const}\)) folgt:

\[ \vec x^{*}\ \parallel\ \vec V_k \]
Erklärung: Das Neuron reagiert maximal, wenn der Input parallel zum Gewichtsvektor liegt. Man denkt an \(\vec V_k\) als räumlichen Kernel bzw. bevorzugtes Eingangsmuster: „Neuron \(k\) ist auf Muster \(\vec x\parallel\vec V_k\) getunt.“ Genau das ist ein Template-Matching per Skalarprodukt.
Folie 2

Temporal dynamics (Feed-forward-Netze)

  • Für FF-Netze typischerweise viel einfacher als für rekurrente Netze (keine Rückkopplungsschleifen).
  • Jedes Neuron ist ein Tiefpassfilter seines Inputs \(V_k(t)\):
\[ \Big(\partial_t + \tfrac{1}{\tau_k}\Big)z_k = \tfrac{1}{\tau_k}\,F\big(V_k(t)+b_k\big) =: f_k(t) \]

Ausbreitung der Aktivität von Schicht zu Schicht über die Green-Funktion \(G(t)=\Theta(t)\,e^{-t/\tau_k}\):

\[ z_k(t) = \int_{-\infty}^{\infty} G(t-t')\,f_k(t')\,dt' \]

Einfaches Beispiel (\(x\xrightarrow{V}z_1\xrightarrow{W}z_2\), mit \(F_k=[\,\cdot\,]_+\), \(b_k=0\), \(V,W>0\), \(x(t)=I\,\delta(t)\)):

\[ z_1(t) = \frac{I\,V}{\tau_1}\,\Theta(t)\,e^{-t/\tau_1}, \qquad z_2(t) = \int_{-\infty}^{\infty}\Theta(t-t')\,e^{-\frac{t-t'}{\tau_2}}\,\frac{W}{\tau_2}\,z_1(t')\,dt' = \dots = \text{Diff. of exp.} \]
Erklärung: In einem Feed-forward-Netz pflanzt sich ein Puls schichtweise fort, jede Schicht glättet ihn per Tiefpass. Zwei hintereinandergeschaltete Exponential-Filter ergeben — wie schon bei den synaptischen Kerneln — eine Difference of Exponentials.
Folie 3

Competing excitatory populations (WTA)

Zwei exzitatorische Populationen \(z_1,z_2\) (je mit Selbsterregung \(w_{ee}\)) treiben eine gemeinsame inhibitorische Population, die die kombinierte Aktivität wahrnimmt und effektiv eine direkte Inhibition \(w_{ei}=\tfrac1a\) zurückgibt (instantane Antwort). \(\tau=200\,\mathrm{ms}\).

\[ \tau\,\dot z_k = -z_k + F\big(w_{ee}z_k + w_{ei}\,F(z_1+z_2) + V_k\big), \quad F(z)=\exp\!\big(a(z+b)\big),\ k=1,2 \]

Per Phasenraum-Analyse ergeben sich mehrere Szenarien:

InputVerhalten
\(V_1=V_2=4\) (schwach)Drei stabile Fixpunkte. FP1: beide Populationen feuern mit moderater Frequenz; FP2+3: eine Population unterdrückt die andere.
\(|V_1-V_2|>\delta\)Nur ein stabiler Fixpunkt → der Gewinner nimmt fast alle Aktivität.
\(V_1=V_2\gtrsim 4{,}7\)Saddle-node-Bifurkation → schon kleinste Unterschiede im Anfangszustand führen zu einem Gewinner.
Name: „Winner-take-all network“ (WTA)
Erklärung: Zwei Populationen konkurrieren über eine gemeinsame Hemmung. Je nach Input entscheidet das Netz entweder „beide gleich“ (schwacher Input) oder „einer gewinnt“ (starker/asymmetrischer Input). Das ist ein zentrales Motiv für Entscheidungen und Aufmerksamkeit im Gehirn.
Folie 4

Phasenraum: Nullклinen (schwacher Input)

Phasenebene \((z_1,z_2)\) mit Vektorfeld und den beiden Nullклinen \(\dot z_1=0\) (durchgezogen) und \(\dot z_2=0\) (gestrichelt). Ihre Schnittpunkte sind die Fixpunkte. Bei schwachem Input \(V_1=V_2=4\) schneiden sie sich an drei Stellen: einer symmetrisch (nahe \((3,3)\)) und zwei „Ecken“ (Gewinner-Fixpunkte nahe der Achsen).

Erklärung: Nullклinen sind die Linien, auf denen sich jeweils eine Variable nicht mehr ändert. Wo beide sich schneiden, ändert sich nichts mehr = Fixpunkt. Die Pfeile zeigen, wohin sich das System von jedem Startpunkt bewegt.
Folie 5

Szenario 1: symmetrischer Fixpunkt (FP1)

Gleiche Phasenebene, jetzt mit einer Trajektorie (Farbe = Zeit, rot Start → blau Ende). Vom Startpunkt \((7,7)\) läuft das System auf den symmetrischen Fixpunkt nahe \((3,3)\) zu — beide Populationen enden gleich stark.

Erklärung: Bei schwachem, symmetrischem Input und symmetrischem Start gewinnt niemand: Das System landet im FP1, in dem \(z_1\approx z_2\). Es „koexistieren“ beide Populationen.
Folie 6

Szenario 1: Zeitverlauf

Oben \(z_1(t)\) (rot) und \(z_2(t)\) (blau): beide fallen von 7 auf denselben stationären Wert \(\approx 2{,}9\). Unten die konstanten Inputs \(v_{1/2}(t)=4\) (grün gestrichelt, identisch).

Erklärung: Der Zeitverlauf zur Trajektorie von Folie 5: \(z_1\) und \(z_2\) bleiben deckungsgleich und konvergieren gemeinsam — die konkrete Bestätigung des symmetrischen Fixpunkts.
Folie 7

Szenario 2: Winner-take-all (Trajektorie)

Phasenebene mit einer Trajektorie, die vom Start (~\((9,5)\)) in eine Ecke nahe \((13{,}5,\ 0{,}5)\) läuft: \(z_1\) gewinnt und übernimmt fast die gesamte Aktivität, \(z_2\) wird nahezu ausgeschaltet.

Erklärung: Sobald der Startzustand (oder Input) unsymmetrisch ist, zieht das System zu einem der Gewinner-Fixpunkte. Die gemeinsame Hemmung sorgt dafür, dass der Stärkere den Schwächeren unterdrückt.
Folie 8

Szenario 2: Zeitverlauf

\(z_1(t)\) (rot) sinkt zunächst leicht, steigt dann auf \(\approx 13{,}5\); \(z_2(t)\) (blau) fällt auf \(\approx 0{,}5\). Input weiterhin \(v_{1/2}=4\).

Erklärung: Zeitverlauf zur Trajektorie von Folie 7. Trotz gleichen Inputs (\(v=4\)) entscheidet die Anfangsbedingung: \(z_1\) setzt sich klar durch — klassisches WTA-Verhalten.
Folie 9

Szenario 2b: asymmetrischer Input

Jetzt unterschiedliche Inputs: \(v_1=5\) (durchgezogen), \(v_2=3\) (gestrichelt). \(z_1\) steigt auf \(\approx 15{,}4\), \(z_2\) fällt praktisch auf 0.

Erklärung: Wenn der Input selbst asymmetrisch ist (\(|V_1-V_2|>\delta\)), gibt es nur noch einen stabilen Fixpunkt — der stärker getriebene Kanal gewinnt zuverlässig. So selektiert ein WTA-Netz den stärksten Input.
Folie 10

Szenario 3: Saddle-node (Nullклinen)

Phasenebene für \(V_1=V_2\gtrsim 4{,}7\): Die Nullклinen berühren sich jetzt fast tangential — der symmetrische Fixpunkt und ein Sattelpunkt rücken zusammen. Das ist die Situation kurz vor / bei einer Saddle-node-Bifurkation.

Erklärung: Bei einer Saddle-node-Bifurkation verschmelzen ein stabiler und ein instabiler Fixpunkt und verschwinden. Genau am kritischen Punkt reagiert das System extrem empfindlich auf kleinste Störungen.
Folie 11

Szenario 3: Trajektorien

Zwei Trajektorien in dieser kritischen Konfiguration: Beide nähern sich zunächst dem (fast verschwundenen) symmetrischen Bereich nahe \((4,4)\), verharren dort und laufen dann in eine Ecke — der Gewinner steht erst spät fest.

Erklärung: In der Nähe des Saddle-node „klebt“ die Dynamik (slow region): Das System zögert lange nahe dem symmetrischen Zustand, bevor es sich entscheidet. Kleine Anfangsunterschiede bestimmen, welche Ecke gewinnt.
Szenario 3

Trajektorie zum Gewinner (Detail)

Eine weitere Trajektorie startet bei \((7,7)\), läuft zum kritischen Bereich nahe \((4,4)\) und kippt dann entlang der Nullклine in eine Gewinner-Ecke bei \(z_1\approx 15\).

Erklärung: Anschauliche Darstellung des „Entscheidungsmoments“: Erst Annäherung an den labilen symmetrischen Zustand, dann klare Trennung. Das illustriert, wie ein WTA-Netz nahe der Bifurkation als sensibler Entscheider wirkt.
Szenario 3

Langsame Umschaltung (Zeitverlauf)

Zeitachse bis \(t=40\): \(z_1\) und \(z_2\) verharren sehr lange nahe \(\approx 4\) (fast identisch), bis das System bei \(t\approx 33\) plötzlich umschaltet — \(z_1\) springt auf \(\approx 14{,}5\), \(z_2\) fällt auf \(\approx 0{,}5\). Input \(v_{1/2}\approx 4{,}7\).

Erklärung: Das ist die kritische Verlangsamung (critical slowing down) nahe der Bifurkation: lange Latenz im labilen Zustand, dann abrupte Entscheidung. Solche Dynamik modelliert z. B. wahrnehmungsbezogene Entscheidungen mit variabler Reaktionszeit.
Folie 14

Input integration mit mehreren WTA-Populationen

Erweiterung auf viele konkurrierende Populationen \(z_k\), jede auf ein anderes Muster getunt (\(V_k=\sum_i V_{ki}x_i\)), die alle denselben präsentierten Input \(\vec x\) sehen und in einer WTA-Konkurrenz stehen: „Wer passt am besten?“

  • Zuverlässige Detektion von Inputs (durch Konkurrenz).
  • Eindeutiger Output-Code (der Gewinner nimmt fast die gesamte Aktivität).
Erklärung: Kombiniert man Tuning (Folie 1) mit WTA-Konkurrenz, entsteht ein Klassifikator: Jede Population misst per Skalarprodukt, wie gut der Input zu ihrem Muster passt, und die Konkurrenz wählt den besten Treffer aus. Das ist die Grundidee vieler neuronaler Erkennungssysteme.
